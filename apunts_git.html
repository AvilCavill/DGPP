Introducció a Git

Git és un sistema de control de versions distribuït que permet gestionar i controlar canvis en el codi font dels projectes. És àmpliament utilitzat en desenvolupament de programari gràcies a la seva eficàcia i flexibilitat.

Conceptes Bàsics de Git

  • Repositori: Un repositori és una carpeta que conté tots els arxius i l'historial de versions d'un projecte. Git utilitza aquest repositori per gestionar canvis i permetre el control de versions.
  • Commit: Un commit representa un "punt de control" en l'historial de Git. És un conjunt de canvis desats amb un missatge descriptiu per identificar què es va modificar.
  • Stage: El stage (o staging area) és una àrea intermèdia on es col·loquen els canvis abans de fer un commit. Permet preparar els fitxers que es volen incloure en el proper commit.
  • Fork: Un fork és una còpia d'un repositori que es crea en un compte diferent, generalment per col·laborar en projectes d'altres usuaris sense modificar l'original.
  • Master: La branca "master" era la branca principal d'un projecte, però en la majoria de projectes actuals s'ha canviat pel nom "main". És la branca per defecte on es fan els canvis finals d'un projecte.
  • Merge: El merge (fusió) és el procés de combinar canvis de diferents branques en una sola. Es fa servir quan es volen integrar les modificacions d'una branca en una altra.
  • Branch: Una branch (o branca) és una línia de desenvolupament separada. Permet treballar en noves funcionalitats o corregir errors sense afectar la branca principal.
  • Checkout: El checkout és l'acció de canviar d'una branca a una altra o d'un commit específic dins d'una branca. Permet navegar entre diferents versions del codi.

Configuració Inicial de Git

Abans de començar a treballar amb Git, és recomanable fer algunes configuracions bàsiques:

  • git config --global user.name "Nom d'usuari" - Estableix el teu nom d'usuari.
  • git config --global user.email "correu@exemple.com" - Estableix el teu correu electrònic.
  • git config --global core.editor "nom_editor" - Defineix l'editor de text predeterminat.

Principals Comandaments de Git

  • git init - Crea un nou repositori Git en el directori actual.
  • git status - Mostra l'estat dels arxius al repositori.
  • git add - Afegeix fitxers a l'índex (staging area).
  • git commit -m "Missatge" - Desa els canvis amb un missatge descriptiu.
  • git log - Mostra l'historial de commits.

Treball amb Branques

Les branques permeten treballar en línies de desenvolupament separades. A continuació es mostren els comandaments més utilitzats:

  • git branch nom_brança - Crea una nova branca.
  • git checkout nom_brança - Canvia a una altra branca.
  • git merge nom_brança - Fusiona una branca amb la branca actual.
  • git branch -d nom_brança - Elimina una branca.

Repositoris Remots

Per col·laborar amb altres desenvolupadors, podem treballar amb repositoris remots:

  • git remote add origin URL - Associa un repositori remot al projecte local.
  • git push -u origin master - Envia els canvis al repositori remot.
  • git pull - Obté els canvis més recents del repositori remot.
  • git clone URL - Crea una còpia local d'un repositori remot.

Flux de Treball de Gitflow

Gitflow és una estratègia de ramificació per a equips que facilita el treball en equip i la gestió de versions amb Git. En el Gitflow, les principals branques són:

  • Master: Conté el codi de producció final.
  • Develop: És la branca principal de desenvolupament on es fusionen les noves característiques.
  • Feature Branches: Branques per a cada nova característica o funcionalitat.
  • Release Branches: S'utilitzen per preparar una nova versió de producció.
  • Hotfix Branches: S'utilitzen per solucionar errors crítics en producció.

El flux de treball de Gitflow implica crear una branca de característiques a partir de develop, fusionar-la a develop una vegada acabada i eventualment fusionar develop a master per a una nova versió.

Resolució de Conflictes en Git

Els conflictes es produeixen quan dos canvis incompatibles es fan en les mateixes línies de codi. Per resoldre'ls, cal:

  1. Utilitzar git status per identificar els fitxers en conflicte.
  2. Obrir els fitxers en conflicte i buscar les marques de conflicte (<<<<<,=====,>>>>>).
  3. Editar el fitxer per mantenir el codi correcte i eliminar les marques de conflicte.
  4. Fer git add del fitxer corregit i després git commit per completar la fusió.

És important comunicar-se amb l'equip per evitar conflictes i resoldre'ls ràpidament quan apareguin.
